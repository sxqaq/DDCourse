# DDCourse 用户指南

DDCourse 是本地优先的课程播放器。视频从你的设备直接读取，不会上传；播放进度、显示设置和 Web 端笔记保存在当前应用环境中。

## 载入课程

### 选择课程文件夹

适合包含多节课程的视频目录。DDCourse 会递归查找 MP4、WebM、OGG、MOV、M4V、MKV 和 AVI，并按子目录整理成课程合集。

文件夹载入后：

- “选择课程文件夹”变为“刷新”。
- “选择视频”变为“更换文件夹”。
- 桌面端刷新会直接重新扫描原目录并显示新加入的视频。
- 浏览器不能永久保留普通文件选择权限，因此刷新时需要重新选择并授权同一目录。

### 选择视频

适合临时播放一个或多个视频，不保留可刷新目录。拖放视频文件与此模式相同。

## 课程库

子文件夹会形成课程合集。可以搜索课程名称，或启用“未完成”筛选。课程完成度根据已知视频时长和播放位置计算；播放达到约 90% 或自然结束时标记为完成。

课程合集按每行 3 个显示，合集区域最多占用两行；更多合集可在这个小区域内滚动。点击“收起已完成”可以批量隐藏已经看完的合集，也可以右键任意合集选择“收起此合集”。已收起合集会保存在本机，展开“已收起”入口即可恢复。

## 播放控制

- 空格：播放或暂停。
- 左右方向键：后退或前进 10 秒。
- `P`、`[`：上一节。
- `N`、`]`：下一节。
- `Q/W/E/R`：1×、1.25×、1.5×、2×。
- 底部滑块：0.5× 至 3×。
- “人声增强”：通过 Web Audio 动态压缩提升语音可懂度；部分浏览器或媒体环境可能不支持。

播放过程中每前进约 5 秒会节流保存一次进度；暂停、切换、结束、页面隐藏或离开时也会补存。

## 学习地图、笔记和收藏

收藏重点和添加笔记时会记录当前视频时间，可从学习地图跳回对应位置，也可以编辑或删除。

Web/PWA 的笔记和收藏保存在 localStorage。桌面端还维护“文档/DDCourse/学习笔记.json”，启动时会与本地缓存按稳定 UUID 合并。“笔记位置”可保存并在文件管理器中显示该文件。

## 导入、导出与重置

“导出进度”生成带版本号的 JSON，不包含视频、笔记或收藏。“导入进度”会把记录合并到本机，同一视频的导入记录覆盖本机记录，列表、统计和当前播放器会立即更新。

Web/PWA 与桌面端使用相同进度格式。目标设备需要重新选择视频；相对路径和文件大小必须一致才能匹配。刷新或更换文件夹不会清除进度。“重置”只清除当前课程合集的播放进度。

## 字体和字号

点击顶部“Aa 显示”：

- 默认优先 MiSans；设备未安装时自动回退。
- 可切换系统无衬线、衬线和等宽字体。
- 可导入 TTF、OTF、WOFF 或 WOFF2。
- 可选择小、标准、大三档整体缩放。

自定义字体只保存在本机 IndexedDB，不会上传。用户必须自行确认字体的个人、商业、嵌入和再分发许可，详见[字体与版权说明](./FONTS_AND_LICENSING.md)。

## 数据与隐私

DDCourse 没有账号或云同步。清除浏览器站点数据可能删除进度、笔记、收藏和显示设置；卸载桌面应用前也建议导出进度。完整说明见[存储与隐私](./STORAGE_AND_PRIVACY.md)。

## 常见问题

### 刷新后没有发现新视频

确认格式受支持。浏览器端需要重新选择原文件夹；桌面端若目录无权限，会跳过不可读项目并在开发日志中记录。

### 导入进度后仍显示未观看

进度 ID 使用相对路径和文件大小。请确保目标视频的目录结构和文件大小与导出设备一致。

### 字体导入失败

确认文件是有效的 TTF、OTF、WOFF 或 WOFF2，并且浏览器支持该字体内部格式。字体许可本身不会影响技术加载，但仍必须合法使用。

### 数据保存失败

应用会提示本地存储空间或桌面目录问题。请检查磁盘空间、浏览器存储权限和“文档/DDCourse”目录权限，并及时导出仍可读取的数据。
