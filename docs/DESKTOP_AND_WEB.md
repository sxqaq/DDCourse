# 本地浏览器构建与桌面端

项目不再发布官方 Web 站点。用于开发和自动测试的本地浏览器构建与 Electron 仍共用主要 React 界面，但文件权限不同。

| 能力 | Web/PWA | Electron |
| --- | --- | --- |
| 读取视频 | 用户选择的 File 对象 | `file:` URL |
| 记住目录 | 浏览器通常不保留目录句柄 | 记录最后目录路径 |
| 刷新目录 | 重新选择并授权同一目录 | 直接异步重新扫描 |
| 笔记主文件 | 无 | 文档目录 JSON |
| 离线外壳 | Service Worker | 桌面资源包 |
| 应用内更新 | 无 | 用户手动检查、下载和确认安装 |

浏览器端“刷新”需要再次授权是浏览器权限模型的结果。桌面端通过 preload 的有限 IPC API 操作目录；渲染页面不能直接访问 Node.js 或任意文件系统接口。复制本机或 UNC 文件路径同样经过主进程校验，只允许返回最近选择目录以内的真实路径。

桌面更新不会自动下载，也不会在普通退出时自动安装。只有用户点击“重启安装”后才会停止播放器、保存进度和笔记并调用安装器；应用 `appId`、用户数据目录和“文档/DDCourse”笔记目录保持不变。
