# DDCourse 学习进度 JSON 格式说明

本文档描述 DDCourse 1.2.0 使用的学习进度备份格式，供用户手工编写、修改或由其他程序生成可导入的进度文件。

## 1. 文件基本要求

- 文件编码：UTF-8。
- 文件类型：JSON，建议扩展名为 `.json`。
- 推荐文件名：`DDCourse-progress-YYYY-MM-DD.json`。
- 当前格式版本：`1`。
- JSON 必须使用英文半角双引号，不能包含注释、尾随逗号或 `NaN`、`Infinity` 等非标准值。

进度文件只包含视频播放进度和该视频最后使用的播放速度，不包含：

- 视频文件本身；
- 课程笔记；
- 收藏重点；
- 本周学习时长；
- 最近打开的课程文件夹；
- 应用设置。

## 2. 完整结构

```json
{
  "app": "DDCourse",
  "formatVersion": 1,
  "exportedAt": "2026-07-11T08:30:00.000Z",
  "progress": {
    "课程/第一章/01-导论.mp4::104857600": {
      "time": 125.4,
      "duration": 1800,
      "done": false,
      "updatedAt": "2026-07-11T08:29:50.000Z",
      "speed": 1.25
    }
  }
}
```

## 3. 顶层字段

### `app`

- 类型：字符串。
- 必填：是。
- 唯一合法值：`"DDCourse"`。

大小写必须完全一致。其他值会导致整个文件被拒绝。

### `formatVersion`

- 类型：数字。
- 推荐填写：是。
- 当前合法值：`1`。

为兼容 DDCourse 1.2.0 以前导出的备份，该字段可以省略；省略时按版本 1 处理。填写任何其他版本都会导致整个文件被拒绝。

### `exportedAt`

- 类型：字符串。
- 推荐填写：是。
- 格式：可以被 JavaScript `Date.parse` 识别的日期时间，强烈建议使用 ISO 8601 UTC 格式。

推荐示例：

```json
"exportedAt": "2026-07-11T08:30:00.000Z"
```

该字段只表示备份文件的生成时间，不参与视频匹配，也不会决定某条进度是否覆盖本机记录。字段缺失或日期无效时，导入器会将其规范化为 `1970-01-01T00:00:00.000Z`，但不会因此拒绝文件。

### `progress`

- 类型：JSON 对象。
- 必填：是。
- 作用：保存零条或多条视频进度。

对象的每个键是视频 ID，每个值是一条进度记录。空对象 `{}` 是合法的，但导入后不会改变现有进度。

## 4. 视频 ID 的格式

视频 ID 是 `progress` 中每条记录的键，计算方式为：

```text
相对路径::文件字节数
```

示例：

```text
课程/第一章/01-导论.mp4::104857600
```

三部分含义如下：

1. `课程/第一章/01-导论.mp4`：选择课程文件夹时得到的相对路径，路径分隔符通常为 `/`。
2. `104857600`：文件大小，单位是字节。

两部分之间的分隔符必须是连续的英文冒号 `::`。

### 为什么必须准确填写视频 ID

DDCourse 不按视频标题单独匹配进度，而是按完整视频 ID 匹配。因此，只要下面任一项发生变化，应用就会把它视为另一个视频：

- 文件名或相对路径改变；
- 文件被移动到另一层目录；
- 文件大小改变；

手工生成进度时，最稳妥的方法是先让 DDCourse 导出一次进度文件，复制其中已有的视频 ID，再修改记录内容。仅凭文件名猜测 ID 很容易无法匹配。

如果通过浏览器一次只选择多个独立视频，而不是选择文件夹，相对路径通常只有文件名，例如：

```text
01-导论.mp4::104857600
```

## 5. 单条进度记录

每个视频 ID 对应一个对象：

```json
{
  "time": 125.4,
  "duration": 1800,
  "done": false,
  "updatedAt": "2026-07-11T08:29:50.000Z",
  "speed": 1.25
}
```

### `time`

- 类型：数字。
- 必填：是。
- 单位：秒。
- 限制：必须是有限且大于或等于 0 的数字。

允许小数，例如 `125.4`。它表示恢复播放时的目标位置。

应用只有在 `time > 3` 且 `time < duration - 5` 时才会自动跳转到该位置。因此：

- 前 3 秒内不会自动续播；
- 距离结尾不足 5 秒时不会自动续播。

### `duration`

- 类型：数字。
- 必填：是。
- 单位：秒。
- 限制：必须是有限且大于或等于 0 的数字。

它表示应用记录进度时的视频总时长。视频真正加载后，DDCourse 会用浏览器或 Electron 读取到的实际时长更新该值。

当前导入器不会强制要求 `time <= duration`，但手工编写时应遵守该关系，否则进度百分比和续播行为可能不符合预期。

### `done`

- 类型：布尔值。
- 必填：是。
- 合法值：`true` 或 `false`，不能写成字符串 `"true"`。

`true` 表示课程已完成，课程列表会显示完成状态。正常播放时，当播放位置达到视频总时长的 90% 或视频自然播放结束，DDCourse 会把它标记为已完成。

### `updatedAt`

- 类型：字符串。
- 必填：是。
- 格式：必须是可以被 JavaScript `Date.parse` 识别的有效日期时间。

推荐使用 ISO 8601 UTC 格式：

```json
"updatedAt": "2026-07-11T08:29:50.000Z"
```

该字段用于描述这条记录最后更新的时间。当前导入合并逻辑不会比较新旧时间；只要视频 ID 相同，导入文件中的记录就会覆盖本机记录。

### `speed`

- 类型：数字。
- 必填：否。
- 限制：填写时必须是有限且大于 0 的数字。

它表示该视频记忆的播放速度，例如：

- `1`：正常速度；
- `1.25`：1.25 倍速；
- `1.5`：1.5 倍速；
- `2`：2 倍速。

省略该字段时，应用使用当前全局播放速度。虽然导入器接受任意大于 0 的有限数字，但界面正常可选范围是 0.5 至 3 倍，手工编写时建议保持在此范围内。

## 6. 多个视频的示例

```json
{
  "app": "DDCourse",
  "formatVersion": 1,
  "exportedAt": "2026-07-11T10:00:00.000Z",
  "progress": {
    "课程/第一章/01-导论.mp4::104857600": {
      "time": 600,
      "duration": 1800,
      "done": false,
      "updatedAt": "2026-07-11T09:30:00.000Z",
      "speed": 1.25
    },
    "课程/第一章/02-基础概念.mp4::209715200": {
      "time": 2400,
      "duration": 2400,
      "done": true,
      "updatedAt": "2026-07-11T09:58:00.000Z",
      "speed": 1.5
    }
  }
}
```

## 7. 导入时如何合并

导入不会先清空本机全部进度。合并规则相当于：

```text
导入后的进度 = 本机现有进度 + 导入文件中的进度
```

具体规则：

- 只存在于本机的记录会保留；
- 只存在于导入文件的记录会新增；
- 视频 ID 相同时，导入文件中的整条记录覆盖本机记录；
- 不比较 `updatedAt`，即使导入记录更旧，也仍然覆盖；
- 文件中任意一条记录不合法，会拒绝整个文件，不会部分导入。

## 8. 合法与非法值速查

| 字段 | 必填 | 合法示例 | 非法示例 |
| --- | --- | --- | --- |
| `app` | 是 | `"DDCourse"` | `"ddcourse"`、`"Other"` |
| `formatVersion` | 否 | `1`、省略 | `2`、`"1"` |
| `exportedAt` | 否 | ISO 日期字符串 | 不影响记录校验，但无效值会被规范化 |
| `progress` | 是 | `{}`、进度对象 | `null`、数组、字符串 |
| `time` | 是 | `0`、`125.4` | `-1`、`"125"`、`null` |
| `duration` | 是 | `0`、`1800` | `-1`、`"1800"` |
| `done` | 是 | `true`、`false` | `0`、`1`、`"false"` |
| `updatedAt` | 是 | 有效日期字符串 | `"昨天"`、空字符串 |
| `speed` | 否 | `1`、`1.25`、`2` | `0`、负数、字符串 |

## 9. 手工制作的推荐步骤

1. 在 DDCourse 中选择目标课程文件夹。
2. 播放目标视频几秒钟，使应用生成包含实际路径、文件大小和时长的记录。
3. 点击“导出进度”。
4. 复制备份文件，保留原文件作为恢复副本。
5. 使用支持 UTF-8 的纯文本编辑器修改副本。
6. 只修改 `time`、`duration`、`done`、`updatedAt` 或 `speed`；除非明确知道文件属性，否则不要修改视频 ID。
7. 使用 JSON 校验工具检查语法。
8. 在 DDCourse 中点击“导入进度”，选择修改后的文件。
9. 重新选择同一课程文件夹并检查进度是否匹配。

## 10. 常见问题

### 导入提示“这不是有效的 DDCourse 进度文件”

常见原因包括：

- `app` 不是精确的 `DDCourse`；
- `formatVersion` 不是数字 `1`；
- 缺少 `progress`；
- 某条记录缺少必填字段；
- 数字写成了字符串；
- 时间或时长为负数；
- `updatedAt` 不是有效日期；
- JSON 中有注释、中文引号或尾随逗号。

### 导入成功，但课程没有显示进度

这通常不是 JSON 校验问题，而是视频 ID 不匹配。请对照 DDCourse 当前导出的文件，确认相对路径和文件大小完全一致。

### 能否只写一条记录

可以。`progress` 中可以只有一个视频 ID。导入后其他本机记录会继续保留。

### 能否用导入文件删除已有进度

不能。空的 `progress` 不会删除本机记录。要删除进度，请在应用中使用“重置”，或者导入一条 `time: 0`、`done: false` 的合法记录覆盖对应视频。

### 桌面版和 Web/PWA 版能否互相导入

格式相同，可以互相导入。但只有视频 ID 完全一致时才会匹配。不同设备上的相对路径仍可能不同，因此迁移后应实际核对。

## 11. 旧版三段式视频 ID 兼容

DDCourse 1.2.0 及更早版本使用“相对路径 + 文件大小 + 最后修改时间”的三段式 ID，例如：

```text
课程/第一章/01-导论.mp4::104857600::1783758600000
```

从 1.2.1 开始，导入器和本地数据初始化会自动删除末尾的修改时间，将其迁移为：

```text
课程/第一章/01-导论.mp4::104857600
```

因此旧版导出的文件仍然可以直接导入。如果多个旧 ID 迁移后变成同一个新 ID，应用保留 `updatedAt` 较新的记录。旧收藏、笔记和最近播放位置中的视频 ID 也会在读取时转换。
